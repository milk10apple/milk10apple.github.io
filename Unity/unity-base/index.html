Unity基础

发表于 2020-10-25 分类于 Unity 阅读次数： Valine：

预制体

  • 制作预制体一般先新建一个空物体, 然后在空物体中制作 (便于操作)
  • 预制体属性面板选项
    • revert, 重置预制体初始状态
    • apply, 将修改应用到预制体
  • 预制体的嵌套: 预制体没有层次, 如果一个预制体的某一部分是另一个预制体的实例, 那么预制体保存之后, 它们将没有任何关系, 也就是说 预制体没有嵌套
  • 预制体中删除物体之后, 预制体提示”散了”, 但此时再点属性面板的apply, 还是会应用到预制体

资源管理

项目目录

  • ProjectSettings 很重要, 包含所有的项目设置, 比如Input, Tag, Layer等信息
  • Asset 资源文件夹, Unity项目的所有资源都在Asset文件夹下
  • Library Temp 缓存, 可以删除, 不重要

音乐格式

  • AIFF和WAV一般用于游戏打斗音效
  • MP3和OGG一般用于较长的背景音乐

操作面板

Scene视图

  • Gizmoz
    • 3D Icons 可以调整摄像机, 光源等没有实体的物体的图标大小
    • Show Grid 显示网格

工具栏

  • QWERT, 快速切换左上角的5个工具

属性

  • active 控制显示或者隐藏 (勾选后, 脚本还在运行吗?)
  • static 引擎对静态物体有优化
  • Tag 物体的分类, 可以无限自由添加, 用于逻辑判断
  • Layer 物体所在层, 只能有32个, 用于相机渲染, 碰撞, 灯光等
  • 属性面板右上角锁标记, 可以锁定到当前属性, 锁定后选择其他物体不会改变
  • icon, active勾选框左边可以定义一个可见的icon, 用于标记不可见物体

Game

  • 分辨率调整

  • Stats 查看信息, 帧率, drawcall, cpu, 渲染批次等

  • Display 分屏展示, 如果有多个摄像机, 可以分别设置每个摄像机的Display目标, 多屏幕可以使用, 很少用

快捷操作

Scene视图操作

  • 视图导航: 按住鼠标右键, 使用WASD控制上下左右移动, QE上下移动, 移动时按住shift可以加速, 也可以在Scene视图Gizmos旁边的摄像机图标(Scene Camera)中调整移动速度; 也可以上下左右和鼠标滚轮操作
  • 物体定位: 在层级面板中双击物体; 或者层级面板中选中物体, 鼠标在Scene视图中时按下F
  • 物体对齐到相机位置: ctrl shift f; 在GameObject菜单中可以看到

世界坐标系, 本地坐标系

  • 属性面板中展示的Transform属性position, rotation, scale都是本地参数, 相对于父节点的参数

MonoBehaviour

  • 所有组件都继承于MonoBehaviour类, 包括Transform, Camera等

  • 继承于MonoBehaviour的类不能通过new来实例化, 只能通过挂载的方式实例化

    • 代码方式实例化:

  • 只有public修饰的类实例成员变量才会在属性面板中展示

  • 挂载在物体上的是组件类的实例, 属性面板展示的也是实例的实例的共有属性, 类属性不属于实例, 不会被展示

Start/Update/Awake

  • Start仅在Update函数第一次被调用之前被执行 (渲染流程第一帧)
  • Start函数在MonoBehaviour生命周期中只被调用一次
  • Update在每一帧被调用; Update中的计算不能太耗时, 否则导致游戏卡顿
  • Awake在一个脚本实例被载入/挂载时被调用
  • Awake在脚本生命周期中只被调用一次
  • Awake在所有游戏对象(GameObject)被初始化之后调用, 所以可以安全的与其他游戏对象对话, 或调用GameObject.Find等函数搜索它们
    • 场景初始化: 先将所有GameObject添加到层级视图中, 再随机的创建脚本实例并挂载到对应的GameObject上
    • 脚本实例被挂载到对应的GameObject之后调用Awake
    • 动态加载的脚本, 过程类似, 游戏对象先添加到层级视图, 再挂载脚本实例, 再调用Awake, 之后进入渲染流程; (Awake的调用就在AddComponent函数中被调用)
  • 每个GameObject上的Awake以随机顺序执行
  • Awake在Start之前被调用

GameObject

层级视图中的所有结点都是GameObject游戏对象; 预制体也是一个GameObject对象; 系统自带的立方体, 球体, 以及美工制作的FBX游戏模型等, 都是GameObject.

MonoBehaviour中直接可以使用的gameObject对象就是脚本被挂载的游戏对象

常用接口

  • 创建空的游戏对象

  • 所有游戏对象都会自动添加Transform组件, 无法删除

  • gameObject.active 已过时, 不再使用

  • gameObject.activeInHierarchy 只读, 在Scene中是否active(该节点及所有父节点都是active的), 决定物体是否可见 (物体上的脚本运行是否有关?)

  • gameObject.SetActive(false) 设置是否激活(activeSelf值), 也就是该物体属性面板中的勾选与不勾选

    • Deactive一个GameObject, 会禁用该游戏对象的所有组件, 它所有组件的Update函数也会停止调用
    • 组件中的OnEnable/OnDisable会在GameObject调用SetActive()时被调用

  • gameObject.activeSelf 只读, 如果节点的activeSelf为true, 但父节点未激活, 则该节点的activeInHierarchy是false, 并且不可见

  • gameObject.tag 用于读写对象的标签, 用于逻辑判断, 设置的标签值必须先定义

  • gameObject.layer 用于读写对象的层

  • gameObject.name 名称, 层级中的名称

  • gameObject.GetComponent<Camera>() 获取一个挂载的脚本实例, 没有挂载则返回null, 有多个相同的脚本实例, 则随机返回一个

  • gameObject.GetComponentInChildren<>() 在当前节点或者所有子节点找到一个对应的脚本实例, 返回一个随机实例或者null

  • 同样的还有 GetComponents, GetComponentsInChildren, GetComponentsInParent等方法

创建基本类型物体

  • GameObject.CreatePrimitive(PrimitiveType.Capsule) 创建一个基本对象, 返回GameObject实例

查找

  • 这几个查找函数都只返回激活的物体
  • Find 查找名字为name的游戏对象, 如果未找到则返回null
  • 如果name中含有/, 则这个名字会被视作一个hierarchy中的路径名
  • 耗费性能, 不要在每一帧中使用这个函数, 可以在开始使用成员变量缓存结果

向量

即有大小, 也有方向

向量的模: 向量长度

向量标准化: 保持方向不变, 长度变成1

向量加法: 各分量分别相加, 几何意义

向量数乘: 每个分量分别乘以数值, 进行长度缩放, 若数值小于零, 则反向

向量点乘: 结果为标量, x1*x2 + y1*y2 + z1*z2 |a|*|b|*cos(a,b) 或者说是向量a在b方向上的投影长度

向量叉乘: 结果仍然是向量, a x b = c 则向量c垂直于a和b所在的平面, c与a和b都垂直, c的具体方向, 与坐标系有关; 在Unity世界中是左手坐标系, 所以c的方向用左手确定, 左手从a向b握拳, 大拇指方向即为c的方向; 向量叉乘不满足交换律

Vector3

是结构体类型, 值类型

静态

  • Vector3.kEpsilon 精度

  • Vector3.forward 世界坐标的正前方 (0, 0, 1), 类似的还有Vector3.back Vector3.left Vector3.right等

  • Vector3.one (1, 1, 1) Vector3.zero (0, 0, 0)

  • Vector3.Distance(a, b) 两点之间的距离 这里的a, b代表点

  • 插值

    • 将a, b看作点, c = Vector3.Lerp(a, b, 0.2f) 就是从a点向b点走0.2*总长度的距离所在位置的点
    • 但是插值计算只从0到1, 如果传入负数, 相当于0; 传入大于1的值, 相当于1
    • 不光Vector3具有插值, 还有对标量的Mathf.Lerp, 以及旋转的Lerp, 针对颜色的Color.Lerp

实例

  • vector3.magnitude 向量的模
  • vector3.sqrMagnitude 向量模的平方, 一般开方比较耗费性能, 如果只是比较长度, 用平方比即可, 尤其在Update中
  • vector3.normalized 单位向量

还有很多接口, 可以查看文档了解

旋转

属性面板中的Rotation是相对于父节点的旋转, 也就是 transform.localRotation.eulerAngles

欧拉角的理解(x, y, z)

欧拉角旋转有两种: 静态 (绕世界坐标轴旋转), 动态 (绕自身坐标轴旋转)

Unity中的欧拉角理解: 按照世界坐标系z-x-y依次的顺序分别旋转(roll - pitch - yaw); (物体发生旋转后, 再调整旋转, 发现z轴的旋转是自身坐标系, 而x, y则是世界坐标系, 难以理解, 所以全部按照世界坐标系的z-x-y一次旋转的结果比较容易理解)

Quaternion

Unity中底层使用四元数表示旋转

难以理解, 一般转为欧拉角后再操作

  • 两个四元数相乘, 两个旋转操作的融合, 先按照第一个四元数旋转, 再按照第二个四元数旋转, 不满足交换律
  • q(Vector3) = t(Quaternion) * p(Vector3) 将向量p进行t的旋转操作

缩放

UnityEngine.Object

销毁 Destroy

Unity中所有的类都继承自 UnityEngine.Object, 不论是 GameObject, Camera, 还是 Component.

销毁一个GameObject, component 或者 asset

  • 对象会在当前Update循环结束后立即销毁, 或者在t秒后销毁
  • 如果obj是组件, 则从其绑定物体移除并销毁
  • 如果obj是GameObject, 则销毁它自身和所有绑定的组件, 还有所有子物体全部销毁; 这所有的销毁都是在当前Update循环结束后

克隆 Instantiate

克隆从效率上来说比创建一个对象效率要高, 常用于一些完全相同并且数量庞大的游戏对象, 比如发射的子弹

这和Unity中使用ctrl+D一样, 新的游戏对象包含完全一样的子对象, 脚本实例包括参数, 但是和原对象互相独立, 互不影响

一般用于克隆一个制作好的预制体(GameObject)
