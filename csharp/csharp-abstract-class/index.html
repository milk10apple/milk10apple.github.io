C#中的抽象类

发表于 2021-02-21 分类于 C# 阅读次数： Valine：

抽象类

  • 抽象类使用关键字abstract修饰
  • 不能被实例化
  • 可以包含抽象方法
  • 继承抽象类的非抽象类必须重写其中的其他抽象方法

抽象函数

  • 使用abstract修饰的方法
  • 只能在抽象类中申明
  • 没有方法体
  • 不能是私有的 (public/protected)
  • 继承后必须使用override实现

抽象方法与虚方法(virtual)的区别:

  • 虚方法可以选择是否写逻辑, 抽象方法没有方法体
  • 虚方法在子类中可以选择重写; 但抽象方法在子类中必须实现
  • 虚方法可以在子类中使用base调用

如何选择普通类还是抽象类

  • 不希望实例化的类, 相对比较抽象的类
  • 父类中的行为不太需要被实现, 只希望子类定义具体的规则

抽象类主要在系统整体框架设计时使用
